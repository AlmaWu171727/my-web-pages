七条设计原则
01
唯一主状态
同一时间只输出一个核心矛盾，不并联
02
主策略定方向，修正策略定形态
两层不互相覆盖，职责边界清晰
03
申论强制干预，两级预警
系统主动引入，不等用户，90天温和+60天强制
04
情绪模块是默认配置，不是特殊功能
56%用户高焦虑，情绪干预面向全体用户，不依赖异常信号触发
05
置信度分级 + 硬边界不劝说
新用户保守输出，老用户精准；A4约束只绕开，不试图改变用户
06
面试独立建轨
笔试标签保留，面试独立画像/进程/状态/资源
07
反馈闭环归因，防过拟合
快慢两速闭环，区分策略/用户/噪声三类原因

七层详解
点击各层展开查看完整内容
层 1
数据层
原始输入，持续采集，系统感知用户的全部来源
+
常规采集
  • ·入学问卷（AI对话形式）
  • ·评测得分（各题型正确率/时间）
  • ·学习行为日志（完成率/时长/完播率）
  • ·对话记录（情绪词频/意图识别）
  • ·错题数据（选了哪个错误选项）
行为信号采集
  • ·任务包内停留时间戳
  • ·打开即关（＜60s）→ 动力信号
  • ·做到一半停（＞5min后停）→ 外部打断信号
  • ·日历事件（报名日/考试日）
  • ·用户主动反馈
层 2
标签层
用户画像，A/B/C/D四组共19维，分层更新
+
A · 画像层（月更）
  • A1时间结构（在职/在职+带娃/全职/在校）
  • A2备考经历（首次/2-3战/4战以上）
  • A3目标定义
  • A4硬边界约束
  • A5动机强度
  • A6T0能力快照
  • A7学科先验
B · 执行类型（月更）
  • B1执行节奏（碎片<30% / 节点30-60% / 系统>60%）
  • B2学习风格
  • B3情绪敏感度（默认中高敏感，56%用户高焦虑）
  • B4反馈响应模式
C · 阶段层（周更）
  • C1宏观进程【主控】
  • C2行测学科进程
  • C3申论进程【独立】
  • C4时间紧迫度【覆盖】
层间优先级：C4 ＞ C1 ＞ C2/C3
D · 信号层（实时）
  • ·行为退缩
  • ·成绩骤降
  • ·执行落差
  • ·情绪危机
  • ·阶段跃迁就绪
  • ·日历触发
  • ·SA1两级预警
SA1申论预警采用两级机制：距考试≤90天且申论未启动 → 前端温和提示，不打断行测；距考试≤60天且申论未启动 → 强制插入申论启动包（覆盖当日行测）
D层信号过滤规则：在职型用户周六日完成率高属正常周末节奏，排除在"爆发冲刺"信号之外，不触发激励推送
置信度规则：新用户标签来自问卷估算（保守策略）；老用户标签来自行为计算（精准策略）
数据校准（4006人）：用户构成——在职76%（其中带娃37%）/ 全职15% / 在校8%。备考次数——首次21% / 1次18% / 2-3次25% / 4次以上36%。在职型时间上限建议1~1.5h（普通在职），带娃在职型建议上限1h。
B1执行节奏数据校准：近30天0活跃占62%，1-3天17%，4-7天9%，15+天仅5%。真实活跃度远低于预估，碎片依赖是绝对主体，阈值从45%下调至30%。
层 3
状态层
当前最卡的问题是什么，唯一主状态输出
+

判定规则优先级：信号覆盖状态 ＞ C4时间紧迫度 ＞ C1宏观进程 ＞ C2/C3微观进程

S15行为退缩触发条件修正（数据驱动）：由"3天完成率<20%"改为"连续5天未登录"。原阈值在62%用户近30天0活跃的现实下会产生大量误触发，未登录信号比完成率更准确反映真实退缩。
S6瓶颈题型特殊规则：数量关系需独立判断。49%用户态度是"基本放弃/全靠蒙"——这是真实应试策略，系统需支持"放弃数量关系、时间让给其他模块"的策略分支，不强制补数量关系。
正确率分层校准（基于3221条真实数据P35/P60/P80分位）：未接触/方法未建 <45% · 初建 45-60% · 巩固 60-72% · 冲高 >72%
S1 认知空白 S2 方向迷失 S3 方法未建 S4 知识碎片 S5 执行脱轨 S6 瓶颈题型 S7 速度瓶颈 S8 申论瓶颈 S9 高原期 S10 实战未稳 S11 心态失稳 S12 查漏补缺 S13 末期保温 S14 末期危机 S15 行为退缩 S16 跃迁就绪 SA1 申论未启动 SF1 框架空白 SF2 内容空洞 SF3 表达失控 SF4 候补焦虑
层 4
策略层
主策略定方向，修正策略定形态，两层叠加不互相覆盖
+
主策略（状态决定）
  • ·核心目标：当前要解决什么
  • ·资源主配比：六类资源默认比例
  • ·任务形态：什么样的任务包
  • ·三轨各自有独立主策略
  • ·配比规则随进程动态调整
修正策略（画像决定）
  • A1→ 任务粒度上限 + 触达时机
  • A2→ 起点深度 + 纠偏优先级
  • A4→ 硬边界，不可逾越
  • B1→ 计划密度 + 推送节奏
  • B3→ 情绪干预阈值
  • B4→ 激励机制设计
行测/申论配比规则：启动~系统期 行测70/申论30 · 强化期 行测55/申论45 · 冲刺末期 行测50/申论50
情绪模块策略调整（数据驱动）：情绪干预不依赖"情绪危机信号"才触发——56%用户持续高焦虑，情绪缓冲应内嵌于每日任务包收尾（高掌握度轻题收尾）和每周节点检查，而非事后干预。
数量关系放弃策略：当A7学科先验或C2进程显示数量关系极弱（正确率<35%）且用户明确放弃意愿时，允许进入"数量关系最小化策略"——只保留最易得分的简单题型，其余时间让给言语/判断/资料。
层 5
资源层
底层库 × 交付形式 × 任务包组装，内容与形式解耦
+
内容架构（五层）
  • ·学科：行测 / 申论
  • ·板块：言语/数量/判断/资料等
  • ·模块：按题型划分
  • ·知识簇：按考点聚合
  • ·母题：知识点+考法+模型
人定义标准，AI识别。母题是最小且唯一的知识单元。
交付形式（三类）
  • 认知课程/记忆卡片/积累文档/对话点拨
  • 训练训练题/限时组/写作练习/全卷模拟
  • 复盘复盘对话/批改点评/错题本/情绪干预
情绪干预是唯一不调用内容库的交付形式
任务包组装四步
  • ①选母题（状态+掌握状态+高频+错题优先）
  • ②定形式（掌握状态×主策略配比）
  • ③定难度和时长（冷启动期低一级起步，D14后按行为数据校准；正常期留20%缓冲）
  • ④定顺序（错题首/易到难/交替/成功收尾）
层 6
动作层
推送 / 对话 / 展示 / 系统边界告知，全程无人工参与
+
推送动作（前端）
  • ·每日任务包（定时，按A1时机）
  • ·日历节点推送（事件触发）
  • ·阶段升级仪式
  • ·SA1申论强制推送
对话动作（AI，48h冷却）
  • B1归因对话
  • B2情绪干预对话（内嵌：每日收尾轻触 / 触发：危机信号后全面介入）
  • B3复盘对话
  • B4诊断对话
  • B5激励对话
  • B6申论批改对话
SA1申论强制推送顺序：先由AI发起解释对话（「为什么现在必须开始学申论」）→ 用户确认理解 → 再推申论启动任务包。不直接覆盖，先让用户理解再接受。
情绪危机对话结束后：当天只推1道高掌握度的熟悉题型（约5分钟），以成功体验收尾，保护次日动力。次日恢复正常任务包。
考前内容时机：考前两天推送考场注意事项与准备清单；考前一天只推激励对话，无任何需要记忆或执行的内容。
系统边界——AI能力到头时
  • 触发连续3轮情绪干预无效 / 明确放弃表述 / 主动要求真人陪伴
  • 做停止干预，明确告知"建议和身边信任的人聊聊"
  • 不做假装能解决 · 继续推学习任务 · 给心理诊断 · 做任何保证
  • 系统暂停所有学习推送，保持对话入口开放，等待用户主动重启
层 7
反馈闭环
结果采集 → 归因 → 回写 → 重判，快慢两速
+
归因三类
  • 策略vs同类用户历史基准
  • 用户行为数据（时长/完成率）
  • 噪声单次异常，连续3次才判趋势
回写两层
  • 用户标签更新，快闭环每日
  • 系统全局参数，慢闭环月度
  • 审核系统维度参数需人工审核后生效
防过拟合
  • ·难度下限由进程阶段保底
  • ·每7天强制插入高一级母题
  • ·S9高原期打破，不顺应
  • ·连续3次才判趋势，不因单次调整
  • ·数量关系放弃策略的回写：确认放弃后该模块不再出现在主任务包，仅保留简单得分题维持手感
04
进程架构
三轨并行，笔试后分叉为面试轨或复盘再战轨
行测轨
  • ·启动期：建框架 + T0评测
  • ·系统期：题型方法建立
  • ·强化期：弱项突破 + 速度
  • ·冲刺期：全卷模拟
  • ·末期：保温，停新知识
申论轨（强制启动）
  • 90天第一级温和预警
  • 60天第二级强制插入
  • ·审题理解阶段（概括题要点找不全）
  • ·结构论证阶段（综合分析逻辑不清）
  • ·语言表达阶段（口语化→规范化）
  • ·动笔突破：46%用户畏惧动笔，此阶段强制输出优先
  • ·冲刺：全卷综合模拟
面试轨（笔试后分叉）
  • ·独立画像采集
  • ·7~21天压缩进程
  • ·认知建立 → 框架建立
  • ·模拟实战 → 临考冲刺
  • ·SF1~SF4专属状态
  • ·时间紧迫度以天为单位
笔试结束后分叉逻辑
  • 未进面状态重置为S2方向迷失（含历史数据）· A2更新为多战 · 优先触发上次卡点归因对话
  • 进面试笔试标签全部保留备用 · 启动面试专项画像采集 · 资源池完全切换至面试模块
05
冷启动路径
D0到D14，标签从零到可用的最短路径。最高流失风险期。
D0
注册当天——采集A层初始标签
AI对话问卷（≤8问，不超过10分钟）→ T0评测（强制，不可跳过）→ 个人路径预览
产出：A层初始标签（低置信度）+ A6 T0快照（高置信度）+ 初步状态判定 S1或S2
关键：路径预览消除方向焦虑，是留住用户的关键动作
D1—D3
第一个任务包——保守输出，验证假设
任务量按A1估算值60%下发 · 难度强制从低一级起步，不完全信任T0单次评测结果
后台观察：实际完成率 vs 估算，校验A1标签准确性
完成率＞85% → 任务量可上调 · 连续5天未登录 → 触发S15行为退缩（注：完成率低不等于退缩，停留时间戳可区分"外部打断"和"主动放弃"）
D7
第一次校准评测——标签置信度上升
校验A6基线 · 更新C2/C3进程 · 开始计算B1执行节奏偏好
部分标签从「问卷估算」升级为「行为计算」
D14
冷启动结束——切换正常模式
A层核心标签均有数据支撑 · B1有至少7天行为数据 · C层进程已初始化
满足以上条件 → 从「保守模式」切换到「正常模式」，策略激进程度可上调
06
场景模拟
完整链路验证——基于4006条用户数据最典型画像：在职+带娃，4战以上，距考试90天
用户画像（数据最典型：76%在职 × 37%带娃 × 36% 4战以上）
  • A1在职+带娃型：工作日约1h，周末碎片（家务间隙），时间上限1h/天
  • A24战以上：有历史成绩锚点，旧方法固化，心理疲惫是隐患，需纠偏优先
  • A5主动型动机，但高焦虑（56%用户此类），情绪模块为默认配置
  • A7文科背景，申论有基础但语言口语化；数量关系接近放弃状态
  • B1碎片依赖型（近14天活跃率约25%，低于30%阈值）
  • B3高情绪敏感，模考失利易陷入自我怀疑（数据：33%用户此类）
时间节点 系统动作 状态判定 结果 / 说明
D0 AI问卷（≤8问）→ T0评测 → 路径预览 S2 方向迷失 4战以上：问卷识别旧方法固化，纠偏优先级设高。路径预览重点："你的申论语言需要升级"
D1—D3 上限1h/天，单次≤20min任务包（带娃型粒度） 持续观察 D2带娃未打开产品，D3停留8分钟后关闭。停留时间戳判定：外部打断而非放弃，不触发S15
D7 校准评测 + B1执行节奏首次计算 S3 方法未建 正确率51%（初建区间），判断方法未建。B1=碎片依赖型（活跃率22%<30%），任务粒度维持≤20min
D14 切换正常模式，SA1第一级温和预警触发 S4 知识碎片 申论0使用，前端温和提示（不打断行测）。数量关系正确率32%，启动放弃策略评估
D30 数量关系放弃策略确认，申论SA1第二级 SA1 + S6 用户确认放弃数量关系复杂题型，时间重分配至言语+资料。SA1先AI解释对话，用户接受后推申论启动包
D50 申论进入结构论证阶段，情绪内嵌检查 S8 申论瓶颈 申论语言口语化严重（B3高敏感），每日任务包末尾内嵌情绪轻触：一句激励+高掌握度收尾题，不等危机触发
D65 连续5天未登录 → S15触发归因对话 S15 行为退缩 AI主动发起对话："最近5天没见到你，发生什么了？"归因为带娃+工作双重压力（外部干扰），不判定心理退缩。重启任务量压缩至原计划40%
D80 末期保温，停新知识 S13 末期保温 考前两天推注意事项，考前一天纯激励对话。4战以上用户特别处理：激励对话聚焦"你已经比任何时候准备得更充分"
07
落地调研
从思想产品到可落地产品，这些信息不调研就无法做决策
已有数据基线（4006条用户画像 + 163份问卷）：用户构成、备考状态、情绪分布、竞品不满意点、行测/申论薄弱点已有真实数据。以下调研聚焦数据仍无法回答的问题。标注「✓ 已有数据」的条目说明设计假设已被验证或校准。

以下按"现有数据无法回答、不调研就无法做决策"的紧迫程度排序。

第一优先级——不调研就无法定产品方向
1
商业可行性——用户愿不愿意为这个产品付钱
用户愿意为"个性化策略"付费的意愿未验证。竞品口碑数据已有（52%不满意"泛泛不针对薄弱"），但付费转化数据仍缺失
最先调研
核心问题

  • ?愿意付费的用户和不愿意付费的用户，在什么维度上有本质差异（备考阶段、职业背景、考试次数）？
获取路径（具体怎么做）

  • →去粉笔、华图App的评论区，找抱怨"不知道自己哪里弱"、"计划不适合自己"的评论——这些是潜在付费点
  • →在公考论坛（公考鸟、微博#公考#话题）发帖：你们都用什么工具？用过哪些觉得值得钱？哪些买了后悔？

用户真实备考行为——系统里所有时间假设的来源
✓ 已有数据：用户构成（在职76%）、大致时长分布。仍缺：精确的工作日时长（自报偏高）、流失节点分布、申论启动时机
最先调研
核心问题
  • ✓在职用户构成已知（76%在职，其中37%带娃）。仍需：精确工作日时长（建议让用户打开屏幕时间截图验证）
  • ?用户在第几天最容易流失？（现有数据有注册时间和最后活跃时间，需进一步计算存活曲线）
  • ?申论真实启动时间：现有数据无此字段，SA1两级阈值（90天/60天）仍是估算
  • ✓备考次数分布已知：首次21% / 2-3战25% / 4战以上36%
  • ?4战以上用户的心理状态和具体固化错误，需深度访谈（现有数据仅有问题标签，无具体内容）
获取路径（具体怎么做）
  • →访谈时不要直接问"你每天学几小时"——用户会说理想值。改问：你昨天几点下班？回家后做了什么？学习是从几点开始到几点？被什么打断了？让用户还原昨天，而不是描述习惯
  • →请用户打开手机屏幕时间，找公考相关App的每日使用时长——这个数据比自报准确得多
  • →在粉笔等App的评论区，找"坚持不下去"、"学了几天就放弃"的评论，统计放弃时间节点的分布
  • →找5个二战用户，重点问：你第一次备考时怎么学的？这次有什么不同？你觉得上次失败的原因是什么？
调研结果用来决定什么
  • →A1时间结构标签的取值区间——在职型上限到底是1小时还是1.5小时还是45分钟
  • →冷启动期的任务量保守程度——现在是60%，是高是低？
  • →SA1申论预警的触发时机——90天第一级是不是合理的？
  • →流失风险最高的节点，决定产品早期重点投入在哪里
深度访谈 12~15人（含屏幕时间截图） 竞品评论区放弃节点分析 公考论坛"备考日记"帖子分析

母题体系的建库成本——整个内容架构能不能跑起来
✓ 已知薄弱点分布（数量关系49%放弃、申论畏惧动笔46%）。仍需：母题数量量级估算和单题标注成本
最先调研
核心问题
  • ?行测全部题型加申论，大概有多少个母题？（量级决定工程大小）
  • ?标注一个母题（六维标签+解题路径+错误模式+难度系数）需要多少专家时间？
  • ?知识点/考法/模型三维能不能事先穷举定义？还是每道题都要专家现场判断？
  • ?华图、粉笔的题库已经有标签体系，和母题体系差距多大？能不能基于他们的数据改造？
  • ?AI辅助标注能把人工工作量压缩多少？标注错误的容忍率是多少？
获取路径（具体怎么做）
  • →找1个行测资深老师，让他把言语理解"主旨题"这个模块的所有母题列出来——这个实验本身就能告诉你母题的数量量级和穷举的难度
  • →拿50道真实历年真题，请2个老师独立标注，比对结果——一致率高说明标准可定义，一致率低说明主观性太强，母题体系有问题
  • →联系粉笔或腰果的前员工（领英可以找到），问他们题库标注的实际成本和质量控制方式
  • →用GPT-4跑一批真题的自动标注，人工审核准确率——这个实验一周内可以完成，直接告诉你AI能帮多大忙
调研结果用来决定什么
  • →MVP的内容范围：是先做一个板块（如言语），还是一开始就要全科覆盖
  • →人力和时间投入：需要几个全职内容编辑，建完基础库需要多少时间
  • →如果标注成本太高，是否要调整为更粗粒度的内容架构
专家标注实验（50题 × 2人） AI自动标注准确率测试 竞品前员工访谈

调研执行路线图
  • 第1步第1~2周 · 商业可行性 + 用户真实行为同步推进。用户访谈各10人，同步爬取竞品评论。这两个调研可以合并——一次访谈里同时问付费意愿和备考行为。
  • 第2步第3周 · 母题标注实验。找1个公考老师，一起标注50道真题，同时跑AI自动标注对比。一周内能拿到建库成本的量级估算。
  • 第3步第4周 · 技术可行性评审 + 竞品深度体验。把前三步的发现整合进来，和工程师一起重新审视系统设计，识别需要简化的地方。
  • 第4步第5~6周 · Wizard of Oz 测试 AI对话。基于前四步调整过的设计，用真人模拟AI做一周对话测试，拿到第一手用户反馈。
  • 结论六周调研结束后做一次完整的设计文档更新，再进入原型开发。避免在假设上继续叠加假设。
