系统架构 · 对话是备考引擎
系统拆为三层：对话主引擎负责驱动备考关系与学习推进，能力层负责解决当前问题，画像修正层在通用主逻辑之上修正不同用户的进入方式、推进策略、风险敏感点与表达方式。
关键边界（每个模块"是什么"与"不是什么"）
对象核心定义不是
入口通过多条路径构建目标共识、规划共识与学习模式共识，把用户带入可持续备考关系固定开场脚本、一次性问全信息
调度器围绕当前主矛盾确定本轮主任务，再组织能力组合与推进强度按菜单发内容、按课表机械推进
干预器当风险已让常规推进失效时，优先恢复可对话、可判断、可执行状态所有情绪都要处理的安慰模块
画像修正层在通用逻辑之上，修正进入方式、推进策略、风险敏感点与表达方式独立第四套系统、实时运行的唯一判断依据
能力层被对话引擎调用来解决当前问题的手段集合独立主导对话的功能模块
能力层：六大能力定义与边界
能力不是独立运行的模块，而是调度器为完成当前主任务所调用的解决手段。每个能力有明确的输入、输出和使用场景。
能力定义典型输入典型输出不是
规划 根据用户当前状态和目标，生成可执行的学习安排 目标、时间窗口、水平、可用时间 阶段计划、每日/每周任务、节奏安排 一次性给完所有计划；脱离状态的固定课表
资源 根据当前主任务和卡点，匹配并推送合适的学习内容 当前卡点、学习阶段、可用时长 课程、题目、专项训练、碎片材料 按目录全量推送；不区分用户状态的统一推荐
反馈 对用户刚完成的学习动作给出即时回应，强化学习循环 刚完成的练习、做题结果、学习行为 正确率反馈、进步确认、错题归因、下一步建议 延迟的阶段性报告（那是复盘）；单纯的鼓励话术
测评 通过结构化测试诊断用户当前水平和薄弱点 需要诊断的科目/模块、用户当前阶段 分数、正确率分布、薄弱模块定位、与目标差距 日常做题（那是资源）；全面考试模拟（仅冲刺期使用）
复盘 对一段时间的学习过程做回顾性分析，校正方向和方法 一段时间的学习数据、完成率、正确率变化 阶段进展总结、方法有效性判断、路径调整建议 即时反馈（那是反馈）；情绪安抚（那是情绪支持）
情绪支持 在常规推进中对轻度情绪波动做承接和调节 用户表达中的情绪信号（轻度焦虑、小挫败） 情绪确认、正常化、轻量认知调整、鼓励 干预器的情绪承接（那是风险已升高时的专项干预）；心理咨询
关键区分：情绪支持（能力层）处理的是调度器范围内的普通波动——用户还愿意学，只是情绪有些低。干预器的情绪承接处理的是常规推进已失效的风险状态——用户已经接不住任何任务。前者是推进中顺手做的事，后者需要暂停推进专门处理。
用户生命周期：三个模块如何交替主导
一个用户从首次进入到持续备考，入口、调度器、干预器在不同阶段交替主导。以下是典型的生命周期流转。
一、入口
定义：入口，是对话引擎在首次接触、重新进入或方向重置时，通过识别用户当前状态，建立基本信任、降低进入门槛、获取最小必要信息，并逐步构建目标共识、规划共识与学习模式共识，使用户进入可持续备考关系的能力。
入口的本质目标不是"问清楚"，而是"对齐清楚"。入口不是固定脚本；下方动作链是可调用动作，不是必须线性执行的话术流程。入口的完成标志不是"信息采集完毕"，而是"用户接受了老师角色，并愿意开始第一个具体动作"。
上层：入口要达成的三类共识
共识类型核心问题要对齐的内容达成后的结果
目标共识我们要去哪？考试方向、时间窗口、当前起点、当前主要矛盾备考目标从模糊意向变成双方都能操作的明确方向
规划共识我们怎么去？当前最优先的第一步、为什么先做、近期节奏、暂时不做什么用户认可起步路径，知道先做什么、为什么
学习模式共识我们怎么一起走？老师带学关系、系统按状态调整、用户需持续行动、过程中有反馈纠偏用户接受"跟着老师学"的关系，不是随便聊
下层：入口五类动作工具箱
① 接住
让用户愿意继续对话
② 定位
判断当前该走哪条路径
③ 澄清
建立目标共识
④ 对齐
建立规划+学习模式共识
⑤ 承诺
共识转化为行动起点
五类动作是工具箱，不是固定线性流程。不同用户因信任起点、紧迫度和当前状态不同，走不同进入路径。
入口的四类触发场景与进入路径
每类场景的信任起点、紧迫度、信息缺口不同，因此动作优先级和节奏完全不同。
场景 A · 新用户首次进入
信任为零，需要从"你是谁"开始
用户不了解系统、没有预期，可能试探也可能带着焦虑进来。入口的首要目标是让用户愿意开口并交出基本信息。
动作策略画像修正
接住不急着问，先给身份感和确定感："我是你的备考老师，从现在开始陪你往下走。"E类新手→更慢更温和；D类多次考生→不当小白盘问
定位根据首句判断紧迫度和状态。"下个月就考"→快进澄清；"随便看看"→继续接住C类冲刺→跳过寒暄直接定位；B类宝妈→先承认现实约束
澄清获取最小必要信息（考什么、什么时候、大概什么水平、每天能学多久），但以对话形式而非表单A类在职→关注可用时间碎片化程度
对齐给出第一步建议并解释为什么先做这个。让用户感到"有人在带我"而不是"给了我一堆东西"E类新手→先建现实坐标再给方案
承诺确认首个具体动作："那我们今天先做这一件事好不好？"—
场景 B · 回流用户重新打开
带着中断的负面情绪回来，信任需要重建
用户消失过一段时间，回来时可能有自责、羞耻、低意愿。最怕的是被追问"你怎么没学"。
动作策略画像修正
接住
最优先		不提中断、不追问原因。"好久没见，最近还好吗？" 先关心人，不提学习任务。A类在职→正常化中断，"工作忙完全正常"；B类宝妈→"孩子最近还好吗？"
定位判断当前是有意愿但缺抓手，还是意愿本身就很低。前者直接给动作，后者继续接住D类瓶颈→可能带着"又要重来"的无力感，先稳认知
澄清不重新建档，只确认目标是否变化："还是考那个方向对吧？时间有没有变？"—
对齐给一个极低门槛动作重新接回。不恢复原计划，而是"今天只做一件事"A类→"今天5道题就够了"；C类→重建阶段节奏
承诺确认愿意接回一个动作即可，不追求完整计划—
场景 C · 用户目标变化
信任基础在，但需要重新对齐方向
用户已经在系统里学了一段时间，但考试方向、目标地区、考试时间发生变化。
动作策略
接住不跳过。先确认变化原因、情绪状态。"换方向是因为发现更合适，还是原来的太难？"
定位快速判断：新方向的时间窗口、科目差异、已有积累能复用多少
澄清更新目标信息，重建时间线
对齐基于新方向重新给出起步路径，同时肯定已有积累："你之前的基础不是白费的"
承诺确认新方向下的第一个动作
场景 D · 浅层停留，未真正进入学习
人在系统里，但从没真正开始
用户注册了、登录了、甚至聊过几句，但没有完成过任何实质学习动作。停留在"想"而不是"做"。
动作策略
接住不催促。先理解卡在哪——是不知道从哪开始，还是还没下决心。"你之前看了看，有没有觉得哪里可以先入手？"
定位判断是方向模糊（不知道该做什么）还是动力不足（知道但做不到）
对齐直接给一个极小动作。"你现在有5分钟吗？做3道题试试看。"不解释太多，先让行为发生
承诺以完成首个动作为入口成功标志，不追求完整建档
入口与调度器的中断期边界：用户中断后回来，不一定都走入口的完整回流路径。判断标准不只是中断天数，而是上次共识是否仍然有效。如果用户中断8天但目标、计划、学习模式共识都没变，调度器以低强度直接接回即可（断续期处理）。只有当共识已失效（目标变了、计划已不适用、用户对"跟着学"的意愿需要重建）时，才触发入口回流路径。简单规则：中断≤14天且共识未变→调度器接回；中断>14天或共识已变→入口回流路径。
五类动作的成功与失败判断
每类动作都有可观察的成功信号和失败信号。失败不意味着结束，而是触发策略切换。
动作成功信号失败信号失败后策略
① 接住 用户从一句话变成两三句；开始提供具体情况；情绪从散乱变得可对话 用户不说话/只回"嗯""好"；明显敷衍；直接退出 降低压力，给一个零门槛选择题（"你现在更想先看看资料，还是聊聊你的情况？"）；如果持续沉默，不追问，留一句低压出口
② 定位 系统能判断出紧迫度、当前最优先任务、该走哪条路径 信息太少无法判断；用户表达矛盾 用选择题缩小范围（"你大概什么时候考？今年还是明年？"）；不连续追问超过2轮
③ 澄清 考试方向、时间窗口、大致水平已够用，可启动第一步 用户回避回答水平问题；说"不知道" 跳过自评，直接给2道诊断题做摸底；或暂时跳过，先进入对齐阶段
④ 对齐 用户对"先做什么"无明显抗拒；回应"可以""那我试试" 用户反复追问"所以我现在到底该干嘛"；表示"我不确定" 缩小选项："你觉得先做A还是B？"；解释为什么先做这个而不是那个
⑤ 承诺 用户答应开始一个具体任务；完成首个动作（点击/做题/记录） 用户说"我回头看看"然后没有动作 进一步降低动作门槛（"不用做完，先看第一题就行"）；或设定下次继续的节点
入口整体成功标准
入口成功 = 用户愿意继续对话 + 接受老师带教关系 + 认可当前起步路径 + 愿意开始首个具体动作。
四个条件同时出现，才算入口真正完成。任何一个未达成，入口仍在进行中。
二、调度器
定义：调度器，是对话引擎在用户已进入备考关系后，根据用户当前阶段、行为状态、学习表现和心态风险，判断这一轮最优先要解决的问题，并动态调用能力，以推动用户持续学习和有效提升的能力。
CORE LOGIC
识别主矛盾 → 确定主任务 → 调用能力 → 控制强度 → 推动下一步发生
边界：调度器只处理"如何正常推进"；一旦主矛盾升级为掉线风险、强失控或持续行动失联，主导权切换至干预器。
调度器三项核心职责
核心职责要解决的问题关键判断输出
判断当前主任务这一轮最优先解决什么启动学习、维持连续性、提升正确率、做阶段诊断，还是处理偏离风险本轮主任务
选择能力组合为主任务调用哪些能力调规划、资源、反馈、测评、复盘、情绪的哪些；先后和取舍能力调用方案
控制推进强度这一轮该推到什么程度该推进、收束还是减压；给一个动作还是一组；强推进还是轻推进推进深度与收口方式
主矛盾 → 主任务 → 能力组合 → 强度梯度
主矛盾核心问题典型表现主任务默认能力低强度中强度高强度完成标准
启动问题学习还没真正发生目标模糊、迟迟未开始让学习发生规划+资源 启动1个最小动作启动1–2个基础动作给出起步包与顺序 用户知道第一步是什么并愿意开始
连续性问题学习无法持续断断续续、中断频繁让学习持续规划+反馈 接回1个低门槛动作接回本周节奏重建完整节奏 用户重新接上可执行节奏
有效性问题学习没转化为提升做了很多但提分慢让学习更有效测评+复盘+资源 校正1个关键点轻诊断+1个改法完整重点收束 用户知道问题在哪并有校正方向
稳定性问题用户偏离学习轨道焦虑、自责、掉线风险回到可持续状态情绪+轻规划 稳住+给一个抓手稳状态+接一个动作仅在风险下降后 用户回到可对话、可行动状态
主矛盾默认优先级
优先级类型原因
1稳定性问题掉线风险不处理，其他推进无意义
2启动问题还没开始时不应先谈提效
3连续性问题学不稳时先恢复持续性
4有效性问题已开始且稳定时提效才成立
调度器判断依据
维度关注内容识别什么
行为状态是否在学、是否连续、有无中断启动/连续性问题
学习表现正确率、短板、测评结果有效性问题
心态信号焦虑、自责、退缩、失控稳定性问题
时间阶段距考试时间、是否临考优先级与方式
调度器的实时状态×卡点决策矩阵
上下文关系：上方主矛盾框架定义了调度器"如何思考"——识别主矛盾、选择能力、控制强度。下方状态矩阵则是调度器"如何执行"——系统通过行为数据自动识别用户处于哪种状态（活跃/断续/中断/冲刺），再结合卡点类型，直接查表得到本轮对话策略。四种状态本质上是主矛盾的行为信号来源：活跃期对应有效性问题为主，断续期对应连续性问题，中断期触发入口回流，冲刺期叠加时间压力。
状态识别规则（行为数据驱动）
活跃期
近7天登录≥5次；完成率≥60%；刷题量稳定
断续期
7天内登录2–4次；完成率<40%；刷题下滑
中断期
超过7天未登录
冲刺期
距考试≤30天

三、干预器
定义：干预器，是对话引擎在识别到用户出现掉线风险、失控风险或明显偏离持续学习轨道时，暂停或重排常规推进逻辑，优先通过情绪承接、认知校正、行动降门槛等方式，使用户回到可对话、可执行、可持续状态的能力。
干预器不是默认模式，也不是情绪陪伴模块；只有当常规推进已经失效，或继续推进会显著增加中断、退出与失控风险时，才由干预器主导。
干预器三项核心职责
核心职责要解决的问题关键判断输出
识别风险状态当前是否已不适合常规推进普通波动，还是掉线/失控/退出风险是否切换到干预
重建可持续状态把用户从偏离边缘拉回先承接情绪、校正认知，还是降低行动门槛回到可对话/可执行/可持续
交还推进权干预做到什么程度可回归风险是否下降，承接能力是否恢复交还调度器主导
风险层级与启动判断
风险层级典型表现是否启动干预处理方式
普通波动累、慌、做得不好，但仍愿意继续学否调度器内轻承接、轻调整
升高风险学不进去、自我怀疑、抗拒任务、承接能力下降视情况收缩推进，必要时切换干预器
必须干预放弃倾向、强自我否定、强失控感、持续行动失联是干预器主导，暂停原推进逻辑
三类核心干预方式
干预方式定义解决什么典型场景目标状态
情绪承接接住当前感受，降低失控感和防御感用户还能不能继续对话焦虑、自责、回流羞耻、情绪上头可对话状态
认知校正修正因焦虑受挫产生的失真判断用户还能不能稳定看待局面自我否定、绝对化判断、局部失败放大为整体可判断状态
行动降门槛缩小动作、降低难度，让用户重新动起来用户还能不能执行计划过重、低意愿、中断回流可执行状态
干预结束与交还推进权
判断维度结束标准交还后处理
可对话用户能稳定回应，不再明显抗拒或失控可交还，但保持低强度推进
可判断不再强烈绝对化/自我否定，能接受现实判断恢复轻量规划，不直接恢复原强度
可执行愿意接受低门槛下一步，具备基本承接能力以低强度接回学习，逐步恢复
原则：干预终点不是"情绪好一点"，而是"用户重新具备常规推进的承接能力"。交还后不默认恢复原强度，由调度器重新评估。
四、画像修正层
核心定位：画像不是系统实时运行的唯一判断依据，也不是独立于入口、调度器、干预器之外的第四套系统；它作为对话引擎的运行修正层，在通用主逻辑之上，修正不同用户的进入方式、推进策略、风险敏感点与表达方式。
使用边界：画像A–E用维度不统一、不互斥的方式描述用户，适合团队对齐场景设计，但系统不能仅靠标签做实时判断。驱动对话个性化需要两层独立机制：首次收集的用户档案（静态基线）+ 持续更新的行为状态（动态信号）。画像变量用于修正对话引擎的执行方式，不替代调度器的状态判断。
画像如何作用于系统各环节
作用环节画像修正什么举例
入口修正信任建立方式、问题顺序、进入路径D类不当小白盘问；B类先承认育儿约束
调度器修正高发主矛盾、主任务优先级、推进强度A类连续性优先；C类有效性优先但需防burnout
干预器修正风险敏感点、典型掉线原因、优先干预方式D类自我否定高敏；B类双重内疚高敏
表达层修正反馈方式、提醒方式、带教语气A类温暖低压；C类直接明确；D类尊重克制
画像输出变量（系统消费接口）
输出变量要回答的问题作用环节
信任起点用户对"老师带学关系"的默认接受度入口
默认推进阻力最容易卡在开始、持续、提效还是重启入口/调度器
高发主矛盾最容易高频出现哪类主矛盾调度器
高敏风险触发点最容易因什么进入风险状态干预器
表达偏好更适合被怎么说、怎么反馈、怎么提醒全程表达

15%占比		 有照护责任；时间碎片化；"只能趁孩子睡了学" 先承认现实约束，不预设整块时间；建立"碎片也能学"共识 连续性问题为主，启动问题次之 家庭突发导致中断；双重内疚（没陪好孩子也没学好） 理解约束不施压；低强度、可拆分、碎片化推进
C 全职冲刺者
14%占比		 考试时间较近；目标明确；强紧迫感 快速进入目标和路径共识；少寒暄，先给阶段感 有效性为主，临考期稳定性上升 临考慌乱/节奏失控；经济焦虑；周期性burnout 直接明确结构化；轻中到中高强度；更早引入测评复盘
D 焦虑瓶颈者
30%占比		 二战/多次备考；有卡分经历；"学了很久还是不行" 不当小白重新盘问；先承认已有投入和卡点，再切入诊断 稳定性与有效性并存，默认先稳态 自我否定；瓶颈固化；失败经验反复唤起 尊重、克制、不空泛鼓励；先稳认知再进入诊断
E 预期失真新手
15%占比		 首次备考；对投入成本缺乏现实感；"跟着学是不是就能上" 先建立现实坐标，不急着给大方案；先摸底再给第一步 启动问题为主，后续易转稳定性 首次受挫后预期坍塌；把没做好等于没希望 务实清晰去幻想化；低到轻中强度；先摸底再收束
