02 / UPSTREAM CONSTRAINTS

上游约束层

上游约束共同定义可行空间 · 在开始构建之前就已生效

这三个板块不是三个并列的知识模块，而是一条连续的决策链。子系统决定你处于哪个构造宇宙，模数规范将这个宇宙量化为精确的几何约束，色彩系统则在供货、结构与叙事三个维度进一步筛选哪些方案能够成立。三者共同构成设计的可行空间。

01 · 子系统
决定构造宇宙
进入哪个子系统，即进入其完整的接口规范与内部逻辑。子系统间兼容性是有条件的结果，非默认前提。
→
02 · 模数规范
决定几何可行域
将子系统差异量化为精确的接口尺寸与模数约束。几何兼容性由此层决定，不是猜测，是计算。
→
03 · 色彩系统
决定哪些方案成立
在几何可行域内，色彩供货进一步筛选哪些方案可实现；色彩语义约束叙事方向。

01 · 四大子系统

System
经典系统
标准砖块体系，1.6mm基础模数，凸点间距8mm，高度9.6mm（含板）。零件兼容性最广，城市、建筑、人仔系列均基于此。
接口：标准凸点 · 8mm间距 · 1.6mm壁厚
Technic
机械系统
轴、销、梁为核心接口。模数与System共享，但以孔位间距8mm为基础，面向机械运动与结构力学。销孔与凸点不能直接互通。
接口：5mm销孔 / 5.9mm轴孔 / 连杆
Duplo
大颗粒系统
为幼儿设计，凸点直径与间距是System的2倍，高度模数为2倍。单向兼容——Duplo底面可接System凸点，反向不可。
接口：16mm间距 · 单向兼容 · 2× 模数
CCBS
球窝关节系统
源于Bionicle，以球窝接头提供三维自由旋转。骨架件通过轴孔与Technic部分兼容，拓扑组织逻辑完全不同。
接口：球窝 / 5mm轴孔（部分）

02 · 模数规范与跨系统兼容

平面模数
水平方向基准
System与Technic共享8mm水平模数。所有水平组合以此为整数倍约束。非整数倍偏移依赖半模数件桥接。
高度模数
垂直方向基准
System标准砖高9.6mm，板件3.2mm（1/3砖高）。3板 = 1砖是垂直混搭核心规则。Technic梁销孔高度8mm与水平模数对齐。
半模数与偏移
非整数倍桥接
4mm（半砖）偏移依赖SNOT转接块、侧凸点板。SNOT技术将垂直堆叠转为水平拼接，突破标准模数网格限制。
跨系统桥接
有条件的接口转接
每多一层转接引入一个力学薄弱点。System↔Technic需专用插销件。System↔Duplo单向兼容。System↔CCBS需Technic轴。
子系统水平模数高度模数接口类型与System兼容性桥接方式
System 8mm9.6mm / 3.2mm板 凸点 + 管道内壁 基准系统—
Technic 8mm8mm（销孔中心） 销孔 + 轴孔 有条件Technic销插入砖块凸点列
Duplo 16mm19.2mm 大凸点（单向） 单向Duplo底面凹槽接收System凸点
CCBS —（有机形态）—（关节驱动） 球窝 + 5mm轴 独立通过Technic轴作中间层
接口设计原则：跨系统桥接总是有成本的。每多一层转接，就引入一个潜在的力学薄弱点与几何累积误差。应尽量在单一子系统内解决问题，仅在功能明确需要时才引入桥接件。

03 · 色彩系统

色彩是约束，不是装饰 · 纵向贯穿全局，在组件、构建、评价各阶段持续作用

供货约束
颜色可用性由官方零件库决定
并非所有零件类型都有全部颜色
颜色稀缺性直接影响成本与可行性
关键风险：接近完成才发现颜色无法获取
层次规则
内部结构色：功能优先，不可见色
外部表皮色：叙事优先，形成印象
强调色：语义定位，≤20%占比
地面/底板：中性色，不参与叙事竞争
聚焦规则
主色2–3种，构建视觉主轴
强调色指向场景焦点区域
色彩分散 = 叙事失焦
人仔色彩必须与场景产生对比
语义规则
颜色传递阵营/功能/材质信息
同色系 = 同功能或同归属
跨颜色区域 = 场景边界
颜色语义先于形态语义被读取
色彩系统是横向线程，不在此结束——在中游的组件选择、下游的迭代构建与评价标准（E5 视觉一致性、E6 色彩一致性）中持续约束决策。
OUTPUT → 上游约束层输出的是可行空间：可用子系统 · 几何合法域 · 色彩可选集。中游方法层在此空间内展开。
03 / MIDSTREAM METHODS

中游方法层

在可行空间内 · 定义构造方法的连续决策链

上游定义了空间边界，中游在此空间内回答"怎么做"。这四个步骤构成一条连续的方法链，而非四个独立的知识板块：先明确零件的职责分工，再定义如何聚合，然后设定系统成立的底线约束，最后给出在约束内的协作准则。

01 组件分类 有哪些功能角色可以调用 收起 ↑

六类组件不是平行列表，而是有职能分工的系统层级。分类原则是系统职责，而非零件形状。每类内部还可进一步细分为二级规格。

结构件
骨架承重 · 定义形态与比例 · 所有其他组件的依附基础
承重件 — 主梁与立柱，失效=结构下沉
锁定件 — 固定相对位置，失效=模块错位
跨接件 — 跨越空隙，失效=板件弯曲
框架件 — 定义开口边界
悬挑件 — 单端固定，失效=尾端脱落
补偿件 — 填充模数间隙
连接件
接口转接 · 实现角度与系统间桥接 · 解决几何约束
凸点接口 — 8mm网格内水平定位
轴孔接口 — Technic传动核心
夹持接口 — 弹性夹持，力有上限
铰链接口 — 单轴旋转自由度
球窝接口 — 三维自由旋转
摩擦/无摩擦轴 — 外观相似，功能相反
功能件
运动传动 · 实现机械动作 · 依赖连接件定位
传动类 — 齿轮组、差速器
支撑类 — 轮子、脚轮
转换类 — 旋转↔线性运动
储能类 — 弹簧、弹射机构
驱动类 — 电机、气缸
控制类 — 换向器、离合器
表皮件
外观塑形 · 处理曲面与过渡 · 附着在结构件上，不承重
平面化 — 光滑板件，最高经济性
斜面化 — 斜坡砖，角度有离散值
曲面化 — 圆弧砖，多件组合曲线
边缘收口 — 处理暴露截面
缝隙管理 — 半模数偏移技术
细节提示 — 印刷砖、纹理砖
语义件
场景识别 · 触发叙事语境 · 赋予场景可读性
建筑语义 — 门、窗、楼梯
交通语义 — 车轮、引擎盖
工业语义 — 管道、控制台
自然语义 — 植被、岩石件
生活语义 — 家具、餐具
角色件
叙事入口 · 确定场景比例基准 · 引入主体视角
角色主体 — 人仔：1.5砖宽×4砖高，比例基准件
姿态组件 — 赋予身份与状态
装备组件 — 定义行为意图
阵营识别件 — 颜色与图形快速传达归属
色彩线程：组件选择时需同步核查色彩可用性。结构件颜色优先考虑供货广泛度；表皮件颜色需满足上游色彩系统的聚焦与语义规则。
02 组织形式 这些角色如何编组 · 如何从零件演化为场景 展开 ↓

零件不是直接堆成场景的，而是经过四个层级的聚合，逐步获得结构、功能与叙事意义。

L1
零件
单一物理零件，具有特定接口和几何形态。由上游子系统与模数规范定义其可用性。
→
L2
模块
具有独立功能意义的组件组合，如一面墙、一个驾驶舱。模块有明确物理边界和独立拆卸能力。
→
L3
功能区
多个模块构成具有完整叙事意义的区域，如建筑的内外庭院、载具的动力舱与乘员舱。
→
L4
场景
多个功能区构成的完整叙事单元，有明确主题、焦点与可读顺序。角色件在此层激活叙事。
聚合原则：每个层级都应有独立的逻辑意义。模块可独立拆卸，功能区可独立替换，场景可局部迭代。这是中游方法与可逆性约束的衔接点。
03 协作约束 什么情况下系统成立 · 哪些是不能逾越的底线 展开 ↓
约束 = 成立底线 · 违反即系统在对应维度失效

协作成立不是靠默契，而是靠约束。五个维度缺一不可——任何一条失效，构建系统即在对应维度崩塌。

几何约束
所有组件必须在三维空间中无冲突共存，接口尺寸精确匹配，不存在体积穿插。
凸点间距、孔径与管道内径需精确配合，公差以0.1mm为单位
旋转角必须为标准离散值（45°/90°及其整数倍）
SNOT偏移需经计算验证，不能凭视觉估计
跨系统桥接时需核算双方模数是否在接口处对齐
力学约束
结构在重力与外力下保持稳定，连接可靠不脱落，活动件运动顺畅不卡死。
重心位置低于支撑底面积的几何中心，防止倾覆
凸点摩擦力有上限——悬挑超约3-4砖后需加锁定件
齿轮比设计需考虑反向锁死风险
薄板件跨度超过8砖后需在中段增加支撑
视觉约束
整体观感在比例、色彩、形态三个维度须达到内在一致性，叙事指向明确。
角色件（人仔）为比例基准，所有建筑与载具从此推导
主色≤3种，强调色≤20%，否则视觉焦点失散
几何语言不混用——有机曲面与直角几何不宜共存于同一主体
纹理密度从场景前景到背景应逐层递减
语义约束
场景传递的叙事信息需自洽，不存在相互矛盾的语义信号，可被观众正确解读。
语义件类型与场景主题须匹配
色彩语义与形态语义不得冲突（如棕色金属结构）
角色件数量与位置须指向叙事焦点，不能均等分布
场景"可读顺序"（前景→中景→背景）需经设计，非随机
可逆性约束 · 乐高哲学的根本
所有组合都是临时的，所有零件都可无损拆卸，所有方案都可重组演化。这不是妥协，而是乐高体系区别于其他积木系统的核心价值主张。可逆性赋予系统永续生命力，也是下游迭代构建的设计前提。
  • 连接力有上限——扣合力的设计上限确保可手动拆卸
  • 零件无损——每次拆装后零件保持功能完整
  • 重组可能性——任何拆解后的零件集合都可形成新的有效组合
  • 设计终态是假设——任何"完成"都只是当前演化阶段的快照

04 配合规则 在约束内应该如何做决策 · 七条协作准则 展开 ↓
规则 = 设计准则 · 违反导致低效或后期大量返工

约束描述系统边界条件；规则描述在约束内如何有效协同。两者不可混用——约束违反导致系统失效，规则违反导致设计低效。

R01
主从规则
每个组合单元有且仅有一个主结构件作为基准，其他所有组件相对于主件定位，而非互相引用。
设计价值：确保拆解任何单一组件不会导致整体坍塌，降低局部修改的连锁影响。
返工触发：移除某"非主要"组件时，3个以上组件随之脱落
R02
层次隔离规则
结构层、功能层、表皮层之间不直接依赖，允许独立替换各层而不影响其他层。
设计价值：可以在不重建骨架的前提下重新配色或更换外观，大幅降低方案迭代成本。
返工触发：调整表皮件颜色时需拆除超过2层结构砖
R03
接口最少化规则
两个模块之间的连接点数量应最小化（通常2–4个），同时满足力学稳定性要求，避免过度约束。
设计价值：减少模块间的依赖程度，允许两个模块独立拆卸与重组，提高可逆性。
返工触发：分离两个独立模块时需拆除超过8个连接点
R04
关键路径优先规则
构建顺序应优先完成对后续所有步骤有约束的关键路径——通常是地基、主体骨架和比例基准件。
设计价值：避免因后期发现关键结构尺寸错误，导致已完成的大量细节工作全部废弃。
返工触发：细节完成后发现主体骨架存在比例错误，必须推倒重建
R05
模块边界清晰规则
每个功能模块有明确的物理边界，模块内部的组件调整不影响边界外的其他模块。
设计价值：支持并行构建多个子模块，减少协调成本。是可逆性约束在组织层面的具体实现。
返工触发：修改某模块内部时，发现必须同时修改相邻模块
R06
反馈闭环规则
每个构建阶段完成后，必须从观察者角度进行全局验证，将验证结果作为下一阶段的输入，而非单向推进。
设计价值：使每个阶段的局部决策都能受到全局语境的校正，避免局部最优导致全局失调。
返工触发：方案"完成"后首次全局审视，发现多处显著问题
R07
场景焦点一致规则
整个场景的结构设计、色彩分配、语义件放置，须共同指向同一个叙事焦点，不允许多个叙事核心同时竞争观众注意力。
设计价值：确保场景有明确的"第一眼"吸引点，建立稳定的叙事层次，避免视觉混乱。
返工触发：询问他人"这个场景的主体是什么"时，得到3个以上不同答案
OUTPUT → 中游方法层输出的是执行逻辑：有职责分工的组件集合 · 层级化的组织结构 · 五维约束空间 · 七条协作准则。下游输出层将执行逻辑转化为实际构建。
04 / DOWNSTREAM OUTPUT

下游输出层

从局部试做到完整场景 · 并判断它是否成立

上游定义了空间，中游给出了方法，下游回答：在实际构建中，如何将方法转化为场景，并判断这个场景是否成立。这层是闭环而非瀑布——任何验证发现的问题都可以触发对上游的返回。

迭代构建 · 闭环而非瀑布

↺ 非线性 · 各阶段均可反推前序 · 局部完成 → 全局验证 → 修正假设

01
主题设定
确定叙事主题、时代背景、情感基调与核心冲突。主题定义构建方向，决定子系统选择与语义件类型的初始范围。
功能模块完成后，主题可能被精化或替换
02
比例假设
以角色件（人仔）为基准，建立整体尺度假设。确定建筑高度、载具体量、地面范围的初始估计值。这是"假设"而非最终决定，后续构建将持续校正。
主体结构完成后比例感与预期不符，通常需要缩小整体规模
03
子系统选择 · 供件核查
根据主题与功能需求选择子系统组合，同时进行初步供件核查——确认所需颜色版本的关键零件在库存或市场中可获得。此步骤将现实可行性引入设计前期。
关键零件供货不足，需返回调整主题或颜色方案
04
局部模块试做
在进入整体布局前，先构建1–2个关键局部模块，验证所选技法与材料是否能实现预期效果。试做的结论会修正整体方案。
局部试做验证失败，需返回重新选择组织形式或连接方案
05
整体布局 · 结构验证
完成主体骨架，建立整体空间布局。进行结构验证——以力学约束为标准检查承重稳定性、悬挑可靠性与连接强度。结构验证通过后方可进入表皮阶段。
骨架完成后比例失衡（通常偏大），需返回缩减规模
06
细节氛围 · 色彩验证
添加表皮件、语义件与色彩强调。进行色彩验证——检查主色分布是否符合聚焦规则，语义件是否与主题一致，整体色调是否实现了预期叙事编码。
色彩发散或语义矛盾，需局部替换零件颜色或更换语义件类型
07
角色布置 · 叙事验证
放入人仔与动物，进行角色验证——检查角色与场景的尺度关系是否正确，角色位置是否激活了预期的叙事焦点，场景是否已具备"可读顺序"。
人仔放入后空间尺度冲突（如门洞太窄），需返回修改结构件尺寸
08
可逆性验证 · 进入评价
执行可逆性验证——确认主要模块可独立拆卸，无循环依赖。可逆性验证通过后，方案进入"当前阶段完成"状态，并以九项评价指标进行系统判定。
当前阶段完成 ≠ 终态 · 可逆性保证任意时刻均可重新演化

T1

T2
功能实现但结构变脆 → 返回 05
T3

T4

T5

评价标准 · 系统判定，不是审美打分

评价不是主观打分，而是系统判定。九项指标各有明确的判定说明与失效表现，可用于方案自审与迭代改进。任何一项低分都指向具体的返工方向。
系统适配度E1
所选子系统组合是否与主题功能需求匹配。跨系统桥接是否有明确的接口转接方案，而非强行凑合。
使用了某子系统但该子系统的优势特性在方案中完全未被利用。
接口效率E2
连接件的使用是否达到最少化原则：同等稳定性下，使用更少的接口点，避免过度约束。
相邻模块通过大面积凸点全面接触，无法在不破坏双方的前提下分离。
结构稳定性E3
结构在预期展示条件下（包括搬运振动）保持形态完整，无零件自然脱落，重心在支撑范围内。
平放展示时稳定，但竖立时因重心过高而倾倒；或悬挑区域在自重下缓慢下沉。
可逆重构性E4
主要模块是否可独立拆卸与替换，拆卸过程是否不损坏其他模块，所有零件拆卸后是否可无损复用。
拆除某附属装饰件时，需连带移除5个以上的结构件；或零件拆卸后出现变形。
视觉一致性E5
整体比例是否以角色件为基准统一推导；各模块的几何语言（硬朗/有机）是否保持内部一致。
场景中某建筑门洞高度仅为1.5个人仔高度；或相邻模块几何语言相互矛盾。
色彩一致性E6
主色数量是否在3种以内；色彩分布是否符合聚焦规则；内部色与外部色是否保持层次隔离。
从正面观察时可见5种以上不同颜色且无明显主次；或内部结构颜色从接缝处大量渗出。
叙事可读性E7
场景是否有明确的叙事焦点；观察者是否能在3秒内理解场景的主要内容与故事语境。
向5位不同观察者询问"这是什么场景"，得到3个以上互不相关的答案。
零件经济性E8
方案是否在达到效果的前提下使用了最少数量的零件；是否存在可以用更通用零件替代稀有零件的改进空间。
某区域使用了大量稀有专用件，而通过不同组织方式可以用常见件实现相同效果。
模块复用潜力E9
方案中的关键模块是否具有通用性，可在不同场景中被复用或作为其他方案的参考模板。
所有模块都高度定制化，任何模块脱离此特定场景后便无独立意义。
OUTPUT → 下游输出层输出的是当前阶段的完成状态：已迭代收敛的场景方案 · 九维系统评价结果 · 可无损拆解并重组的零件集合。这不是终态。
05 / CONCLUSION · 总结
乐高的本质不是拼装，而是在约束空间内进行模块化组织、可逆重构与场景表达。
上游 · 约束系统
可行空间
子系统、模数规范与色彩系统共同构成先决约束，在构建开始前就已定义合法边界。违反上游约束，方案在物理或叙事层面不成立。上游不是知识储备，是设计前提。
中游 · 构建系统
执行逻辑
组件分类、组织形式、协作约束与配合规则构成方法链。约束（成立底线）与规则（设计准则）明确区分：违反约束导致系统失效，违反规则导致后期大量返工。
下游 · 叙事系统
验证与输出
迭代构建是闭环而非瀑布。评价标准是系统判定而非审美打分。任何完成状态都只是当前演化阶段的快照。可逆性保证整个系统可以在任意时刻重新演化。
乐高不是零件堆积，而是由约束空间、构造方法与验证机制共同构成的可逆重构系统。
任何完成状态都只是当前演化阶段的快照。
